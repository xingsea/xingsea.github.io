SSH 的两种端口转发方式

2025-08-22

核心概念：SSH 端口转发（也称为 SSH 隧道）是一种利用加密的 SSH 连接来安全地传输其他非安全网络协议数据的技术。它就像一个加密的管道，将数据从一个端口安全地传输到另一个端口。

1. 本地端口转发

本地端口转发 是将本地机器的某个端口，通过 SSH 隧道转发到远程网络中的另一台机器的某个端口。

使用场景：当你无法直接访问目标服务器（Server B）的某个端口，但有一台跳板机（Server A）可以访问，并且你也能 SSH 登录到这台跳板机时。

命令格式：

1

ssh -L [本地绑定地址:]本地端口:目标主机:目标端口 跳板机用户@跳板机地址

参数解释：

  • -L: 指定本地端口转发。
  • 本地绑定地址： 可选，通常是 127.0.0.1（只有本机可访问）或 0.0.0.0（所有网卡可访问）。默认为本地。
  • 本地端口： 你本地机器上开启的端口。
  • 目标主机:目标端口： 你最终想要访问的服务器的地址和端口。
  • 跳板机用户@跳板机地址： 你有权限登录的中间 SSH 服务器。

工作流程：

  1. 你在本地机器上执行上述 SSH 命令，建立与跳板机（Server A）的连接。
  2. SSH 客户端会在你的本地机器上监听你指定的 本地端口。
  3. 当你访问本地的 本地端口（如 localhost:3306) 时，数据会被 SSH 客户端加密。
  4. 加密的数据通过 SSH 连接发送到跳板机（Server A）。
  5. 跳板机上的 SSH 服务端解密数据，并将其转发到最终的 目标主机:目标端口（如 192.168.1.100:3306）。
  6. 目标服务器返回的数据沿原路反向传输回你的本地机器。

经典例子：访问远程内网数据库 假设公司数据库 mysql.internal.com（端口3306）只能在公司内网访问。你有一台可以 SSH 登录的公司公网服务器 jump.company.com。你可以这样操作：

1

ssh -L 3307:mysql.internal.com:3306 yourname@jump.company.com

之后，你在本地电脑上连接 localhost:3307，就等于通过 jump.company.com 这台跳板机连接到了内网的 mysql.internal.com:3306。

2. 远程端口转发

远程端口转发 是将远程 SSH 服务器的某个端口，通过 SSH 隧道转发到你的本地网络或本地机器能访问的网络中的另一台机器的某个端口。

使用场景：当你需要让外网的人访问你本地电脑（或你本地网络中的某台机器）上运行的服务，但你本地没有公网 IP（例如在家中的电脑）。

命令格式：

1

ssh -R [远程绑定地址:]远程端口:目标主机:目标端口 公网服务器用户@公网服务器地址

参数解释：

  • -R: 指定远程端口转发。
  • 远程绑定地址： 可选，在公网服务器上绑定的地址。通常需要修改 SSH 服务器配置（如 GatewayPorts）才能绑定到 0.0.0.0。
  • 远程端口： 在公网服务器上开启的端口。
  • 目标主机:目标端口： 从公网服务器角度看，它要把数据转发到哪里。通常是你本地机器（localhost）或其所在局域网的其他服务。
  • 公网服务器用户@公网服务器地址： 你有一台有公网 IP 的服务器。

工作流程：

  1. 你在本地机器上执行上述 SSH 命令，建立与公网服务器（Public Server）的连接。
  2. SSH 服务端会在公网服务器上监听你指定的 远程端口。
  3. 当任何人访问公网服务器的 远程端口（如 public-server.com:8080) 时，数据会被发送到公网服务器。
  4. 公网服务器上的 SSH 服务端会将这个端口的数据通过已建立的 SSH 连接加密发送回你的本地 SSH 客户端。
  5. 你的本地 SSH 客户端解密数据，并将其转发到 目标主机:目标端口（如 localhost:3000）。
  6. 目标服务返回的数据沿原路反向传输回公网服务器，最终返回给访问者。

经典例子：临时暴露本地开发环境 你在本地电脑 localhost:3000 运行了一个 Web 项目，想让外地的同事查看。你有一台公网服务器 public.example.com。

1

ssh -R 8080:localhost:3000 user@public.example.com

之后，你的同事访问 http://public.example.com:8080，公网服务器就会将请求通过 SSH 隧道转发到你本地电脑的 3000 端口，从而看到你的网页。

总结与对比

特性 本地端口转发 (-L) 远程端口转发 (-R)
方向 从本地出发，指向远程网络 从远程出发，指向本地网络
命令发起者 需要访问内部服务的客户端 提供内部服务的主机（通常在内网）
监听端口 监听在本地机器上 监听在远程SSH服务器上
主要目的 让我能访问另一个网络的服务 让另一个网络的人能访问我（或我的网络）的服务
常用场景 访问公司内网数据库、Web服务 暴露本地开发环境、将家中NAS服务临时映射到公网
